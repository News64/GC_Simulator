ダメージ公式：
物理：(A.ATK + A.Inherit_ATK) * (A.Skill_Rate + A.ATK_Buff - A.ATK_Debuff) - (B.DEF + B.Inherit_DEF) * (1 + B.DEF_Buff - B.DEF_Debuff) * Reduction_Rate
属性：(A.WIS + A.Inherit_WIS) * (A.Skill_Rate + A.WIS_Buff - A.WIS_Debuff) * (1 + Attr_Factor) - (B.WIS + B.Inherit_WIS) * (1 + B.WIS_Buff - B.WIS_Debuff) * Reduction_Rate
説明：
A：攻撃方
B：守備方
Inherit_ATKなど：フォース、エントラストからのパラメータ上昇（数値）
ATK_Buffなど：バフからのパラメータ上昇（％）
ATK_Debuffなど：デバフからのパラメータ減少（％）
Skill_Rate：戦闘スキル自身の倍率
Reduction_Rate：通常は0.5、戦闘スキルはメテオインパクトなどの貫通スキルの場合は0
Attr_Factor：Criticalの場合は0.15、Blockの場合は-0.15、他は0

スキルのカテゴリー：（シミュレーターと同じ）
1.　登場スキル
2.　戦闘スキル
3.　戦闘不能スキル
4.　回避スキル
5.　反撃スキル
6.　毒スキル

1.　登場スキル
モンス初登場の時に一度だけで発動する。優先度がある（数値は仮定のものです）。
両方とも初登場の時、順番は「早い方の高優先度スキル＞遅い方の高優先度スキル＞早い方の低優先度スキル＞遅い方の低優先度スキル」です。同じモンスが同優先度スキル複数ありますの場合は、発動順番は習得順番と同じです。
ディープスリープ、ファストスリープによって、登場スキルを不発になるの場合は、「次の相手モンス登場」の時、もしくは「モンスが復活した」の時にもう一度発動可能です。（ラストパスチオンの方は未検証です。）
